# Task: fix-class

**Status**: complete
**Branch**: hatchery/fix-class
**Created**: 2026-03-22 14:41

## Objective

When no classes are configured, a default class of "Overall" was assigned, causing a useless "Class" column to appear in all output tables even for unclassed series.

Fix: use `None`/null instead and suppress the class column from rendered tables.

## Summary

### Files changed

- `src/gpt_racing/classes.py`: When `config.classes is None`, return null values for `class_name`, `class_symbol`, and `class_color` instead of `"Overall"` / `"#ffffff"`.
- `src/gpt_racing/render_tables.py`:
  - Changed `has_classes` detection from `n_unique() > 1` → `is_not_null().any()` in both `render_standings` and `render_race_results`.
  - Removed `"class_display": "Class"` from the `else` (no-classes) branch of both render functions.
- `tests/test_classes.py`: Updated `test_no_classes_config` to assert `is_null().all()` instead of checking for `"Overall"` strings.

### Key decisions

- `_format_class` is still called on null struct values (polars `map_elements` skips nulls by default), so no crash occurs — the `class_display` column is computed but simply not selected for output in the no-classes branch.
- `output.py` needed no changes: `n_unique() > 1` correctly returns `False` for all-null columns (null counts as 1 unique value), so per-class table generation remains suppressed.
- `core.py` needed no changes: `.over("class_name")` rank computation works fine with all-null class groups.
