## Objective

Add the ability to set driver classes (e.g. Pro, Am). Drivers in a class are on the field at the same time, compete for the same points, and their class is rendered as part of the output results (with symbol and color).

## Context

This work spanned three iterations:

1. **feat-classes** (2026-03-08): initial design — identified the need for `DriverClass` config, class resolution logic, and output rendering.
2. **feat-classes-2** (2026-03-09): implemented config models, class resolution, and data plumbing through `core.py`. Did not implement HTML rendering.
3. **classes** (2026-03-21): completed HTML rendering with symbol + color, updated tests.

### Key decisions

- **"Overall" synthetic class**: When `config.classes is None`, every driver gets `class_name="Overall"`, `class_symbol="Overall"`, `class_color="#ffffff"`. Existing configs without classes work unchanged.
- **Class display is symbol-based**: The `symbol` field (short, e.g. `"P"`, `"G"`) is shown in the table with a colored background using inline HTML (`<span style='background-color:...'>`). Full `name` is in the data but not rendered.
- **No per-class championship rankings**: Standings rank all drivers globally. Class is display-only. This was an explicit spec decision.
- **Matching priority**: `user_id` > `name` (exact `display_name` match). ValueError on missing name or duplicate assignment.
- **`class_df` schema**: explicitly `pl.Int64` for `user_id` to avoid type mismatches on join.
- **CLI refactor**: `gpt-racing run <config> <out_path>` (was bare command before).
